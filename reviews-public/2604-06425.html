Paper overview

Neural Computers asks a foundational question: can a single neural network take over the runtime responsibilities of a physical computer — executing instructions, maintaining persistent state, and rendering responsive interfaces — without a separate programmed operating system? The paper introduces the Neural Computer (NC) abstraction and a formal four-condition definition of a Completely Neural Computer (CNC).

The core formalism frames the NC as a state machine whose hidden state evolves through the model's own outputs — distinguishing it from AI agents (which control separate execution environments) and world models (which predict environment state without replacing it). Two prototype implementations are presented, both built by fine-tuning the Wan2.1 video diffusion model:

  • NC_CLIGen: a video model trained on ~824k CLI/terminal video streams plus ~128k Dockerized scripted sessions, evaluated on character rendering accuracy and arithmetic probe completion.
  • NC_GUIWorld: a video model for desktop GUI simulation on Ubuntu XFCE, trained on data splits ranging from 1,400h of random interactions to 110h of goal-directed agent traces.

The paper reports 10 controlled ablation experiments and closes with a roadmap toward fully realized CNCs, organized around four measurable open challenges: symbolic stability, routine reuse, controlled updates, and machine-native semantics.

Key strengths

NC_CLIGen is a verified novel domain contribution. An independent search across approximately 250 literature candidates found no prior work on CLI/terminal video simulation. Character accuracy progressing from 3% (untrained) to 54% over 60k training steps and reprompting raising arithmetic probe accuracy from 4% to 83% are the first numbers of any kind in this domain.

SVG cursor supervision is a decisive training insight. Providing explicit per-frame SVG binary cursor masks as visual supervision raises cursor accuracy from 8.7% to 98.7% (delta = 90 percentage points). Coordinate-only and Fourier-encoded baselines both fail entirely, confirming the result is non-obvious.

The data quality over quantity finding provides actionable design guidance. 110h of goal-directed CUA agent data achieves FVD 14.72 while 1,400h of random interaction data achieves at best FVD 20.37. CUA outperforms 12.7× more data by 69.4% on FVD.

The NC/CNC conceptual framework introduces genuine new formal vocabulary. The four-condition CNC specification and formal state machine notation are not anticipated by competing papers. The framework performs real analytical work: it organizes the ablation results and structures the roadmap.

Transparent limitation disclosure is notably uncommon for an ML systems paper. The paper explicitly labels its vision section as "Position," acknowledges open-loop-only evaluation, and discloses negative training findings. This epistemic honesty lends credibility.

Key concerns and constructive suggestions

The central "runtime" claim is unsupported by any closed-loop experiment. All 10 experiments use open-loop evaluation where oracle ground-truth frames are provided at each step. The defining property of a runtime — that the model's own outputs drive subsequent state — is never tested. To address this, the authors could add a minimal 100-task, 5-step autoregressive evaluation in CLIGen Clean requiring no new data collection or model training — only a modified evaluation loop.

Four directly relevant peer-reviewed papers are absent from the citations. GameNGen (ICLR 2025), DIAMOND (NeurIPS 2024 Spotlight), AVID (ICLR 2025), and DWS (ICLR 2025) all establish the technical design space NC_GUIWorld operates in. To address this, the authors should add all four citations and reframe NC_GUIWorld's contribution as the first systematic application of the GameNGen/DIAMOND paradigm to desktop GUI environments — an accurate and still valuable contribution.

NeuralOS receives one sentence despite being the most directly comparable system. NeuralOS (ICLR 2026) demonstrates 256+ frame long-horizon OS state persistence via hierarchical RNN, directly addressing the open-loop limitation. To address this, the authors could add a formal comparison subsection acknowledging NeuralOS's advantage for long-horizon state persistence.

No variance measures appear anywhere across all 10 experiments. For the paper's most consequential architectural recommendation (injection mode, SSIM delta = 0.006, estimated Cohen's d ~0.06–0.15), the difference may be within noise. To address this, the authors could run 2–3 random seeds for the key ablation tables.

The captioning pipeline driving NC_CLIGen's key result is absent from the released code. This means users cannot reproduce core experiments. To address this, the authors could release the captioning pipeline with prompt templates and the episode packaging system.

What would most improve this paper

1. Add a minimal closed-loop experiment. Highest impact-to-effort ratio. Even a degradation curve showing performance collapse would accurately characterize limitations and establish the measurement framework for future NC research.

2. Add the four missing citations and reframe NC_GUIWorld. Writing-only change that removes the most serious scholarly integrity concern.

3. Add a zero-shot Wan2.1 baseline. A single evaluation run to demonstrate fine-tuning actually surpasses the base model.

4. Add statistical variance reporting. Multi-seed runs for the key ablation tables would validate or qualify the architectural recommendations.

5. Add a formal NeuralOS comparison and disclose code attribution in the paper body. Currently the attribution appears only in the README, not the paper.
