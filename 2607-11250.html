Paper overview

Multi-Agent LLMs Fail to Explore Each Other studies whether LLM agents discover and use useful peers in multi-agent systems. It formalizes Multi-Agent Exploration as a partially observable stochastic game and frames peer selection as an exploration–exploitation problem under latent capability and information differences.

The paper proposes Multi-Agent Contextual Exploration (MACE), an interpretable LinUCB-style selector that uses relational features to estimate the value and uncertainty of consulting each peer. The theoretical analysis connects exploration to capability diversity through regret-style results, while the empirical study examines contextual and parametric diversity.

Experiments include motivating diagnostics, transfer, homogeneous and stronger-model controls, sensitivity analyses, and round-wise results. Together they show a promising pattern in the tested small heterogeneous populations, but the linked repository is currently README-only, and the paper evaluates a narrow slice of the adaptive, dynamic settings described by its broader framing.

Key strengths

Clear motivating failure mode. The two-arm diagnostic makes premature peer commitment visible before the paper introduces its full formalism, giving the problem a concrete empirical anchor.

Interpretable intervention. Relational features combined with LinUCB provide a lightweight, inspectable mechanism for selecting peers and representing uncertainty, making the proposed behavior easier to diagnose than an opaque routing policy.

Complementary experimental coverage. Contextual and parametric diversity are supplemented by transfer, homogeneous, stronger-model, sensitivity, and round-wise analyses, so the method is not supported by a single benchmark comparison.

Useful persistence and theory connections. Frozen exploitation and cross-benchmark transfer probe reuse of the learned selector, while the diversity analysis gives a coherent hypothesis for when exploration should create value.

Responsible scope disclosure. The limitations and broader-impact discussion acknowledges scale, dynamics, and misuse risks, which helps distinguish the controlled evidence from the paper's longer-term vision.

Key concerns and constructive suggestions

The theory is broader than its assumptions. The paper frames an adaptive partially observable stochastic game, but Theorem 1 relies on fixed per-pair linear reward parameters. To address this, narrow the theorem's claim to the stated surrogate or develop an adaptive/non-stationary analysis with explicit conditions and model-fit diagnostics.

Reproducibility is currently incomplete. The linked repository at the checked commit contains only a README saying that code will be released soon, preventing code-level verification of prompts, features, rewards, evaluators, and configurations. To address this, release executable source, an environment lockfile, seeds, raw outputs, checkpoints or API versions, and a one-command smoke test.

The in-context comparison has an interface confound. Free-form router outputs can trigger a predefined routing fallback when parsing fails, so the comparison may partly measure action-interface reliability rather than exploration quality. To address this, give every router the same constrained choice interface, report parse and fallback rates, and include a deterministic LLM router with identical action space.

Inferential reporting is incomplete. Means and standard errors favor MACE in the tested settings, but complete independent-run distributions, formal tests, and effect sizes are not consistently available, while several main effects appear small or uncertain. To address this, release per-task and per-seed outcomes, use paired bootstrap or mixed-effects intervals, report effect sizes, and explain multiplicity handling.

Generality and practical cost remain under-tested. The study uses small populations, short horizons, limited diversity variation, and one transfer pair, without reporting the token, API, latency, or communication cost of exploration. To address this, sweep population size, horizon, churn, delays, task families, capability diversity, and communication budgets while reporting cost per quality gain.

What would most improve this paper

1. Release a complete reproducibility package. Publish source, prompts, evaluator and feature/reward implementations, configurations, seeds, raw outcomes, version information, and a smoke test so the central comparisons can be independently checked.

2. Remove the baseline interface confound. Re-run all routers with a constrained action schema, publish parse/fallback statistics, and add matched delegation, learned-routing, or topology-selection baselines.

3. Establish statistical and practical uncertainty. Provide paired per-task/per-seed data, confidence intervals, effect sizes, formal tests, and calls, tokens, latency, and communication overhead per improvement.

4. Align the theory with adaptive interaction. Either explicitly scope the guarantees to the fixed linear surrogate or analyze non-stationary peers and validate the model assumptions empirically.

5. Test the operating envelope. Vary population size, interaction horizon, capability diversity, context noise, churn, delayed feedback, model families, tasks, and communication budgets to determine where peer exploration remains useful.
