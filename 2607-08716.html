Remember When It Matters: Proactive Memory Agent for Long-Horizon Agents

A clean, plug-and-play memory monitor makes preserved execution state behaviorally useful at the moment of action, with notable reported gains across two complementary benchmarks. The most important next step is to isolate that mechanism from added frontier-model deliberation through matched-compute controls, paired uncertainty estimates, transparent cost reporting, and complete public artifacts.

Paper overview

Remember When It Matters studies a practical failure mode in long-horizon agents: requirements, environment facts, prior attempts, diagnoses, and open subgoals can remain somewhere in a growing trajectory yet stop influencing the next decision. The authors call this behavioral state decay and treat memory as an active intervention mechanism rather than a passive retrieval store.

The proposed Proactive Memory Agent runs beside an unchanged action agent in two phases. It first updates a private structured memory bank from recent trajectory evidence, then decides whether to inject a memory-grounded reminder or remain silent. This separation preserves compatibility with black-box agents and existing harnesses while making the timing of memory use an explicit control decision.

Across Terminal-Bench 2.0 and τ2-Bench, the paper reports pass@1 improvements for both weaker and stronger action agents, including gains of 8.3 percentage points and 6.8 percentage points for Sonnet-based conditions. Ablations compare selective intervention with passive bank exposure, always-on reminders, advisor-only guidance, and general retrieval. An open-weight study also trains Qwen3.5-27B on SETA using supervised fine-tuning and GRPO, showing improved validation reward and partial transfer to Terminal-Bench.

Key strengths

Clear problem-to-mechanism alignment. The structured bank preserves execution state while the intervention decision makes selected information salient when an action is chosen. This is an intelligible decomposition of storage from behavioral influence.

Modular deployment with unchanged action agents. The memory component operates alongside the action policy rather than requiring access to its weights. That makes the design relevant to closed frontier models, existing harnesses, and heterogeneous agent stacks.

Complementary empirical settings and meaningful point improvements. Terminal debugging and policy-governed service conversations exercise different forms of long-horizon state. Improvements across both domains and across action-agent strengths make the system more compelling than a single-benchmark demonstration.

Mechanism-oriented ablations. Comparisons against full-bank exposure, always-on injection, a no-bank advisor, and Mem0 probe whether value comes from structured state, selection, or generic auxiliary guidance. This is the right experimental instinct even though tighter controls remain necessary.

A credible open-weight direction. The Qwen3.5-27B SFT and GRPO study suggests that proactive memory policies need not remain dependent on a closed frontier model. Its partial transfer result provides a useful starting point for a more fully specified training program.

Key concerns and constructive suggestions

The implementation is not presently reproducible. The repository advertised by the paper has no commits, while exact prompts, configurations, task manifests, raw trajectories, training details, and result artifacts are absent. To address this, the authors should publish a tagged implementation with environment lockfiles, data provenance, prompts, configs, task-level outputs, exclusions, and an end-to-end reproduction command.

Point estimates lack statistical reliability measures. The paper reports no confidence intervals, repeated stochastic runs, or paired hypothesis tests, leaving unclear how stable the observed gains are across task and evaluator variation. To address this, the authors should repeat the main conditions, report paired bootstrap intervals and McNemar-style tests where appropriate, and document seeds, run counts, and exclusions.

The causal contribution of selective structured memory is not isolated from added computation. A separate Opus-class model reasons at every step, so a similarly expensive critic or self-reflection module might produce comparable gains without a memory bank. To address this, the authors should run a compute-matched factorial study crossing bank versus no bank with selective, always-on, and silent intervention, alongside equal-token advisor, self-reflection, random-reminder, and no-op controls.

Result tables and selective-superiority wording need reconciliation. Full-system τ2-Bench values differ across tables, and always-on injection leads the Table 2 micro average even though selective intervention has the stronger macro balance. To address this, the authors should regenerate every table from versioned source results, explain population or run differences, and qualify claims by the aggregation under which each method leads.

Deployment cost and latency are unreported. Calling a frontier memory model at every step may materially increase tokens, wall-clock time, and dollar cost, which could change the practical ranking of methods. To address this, the authors should report calls, tokens, latency, cost, bank growth, and error rates, then present quality–cost Pareto curves against matched baselines and less frequent triggers.

Behavioral state decay is named but not directly measured. Benchmark success is consistent with the proposed explanation, but the evaluation does not identify decay events, measure intervention precision and recall, or show when reminders repair or harm decisions. To address this, the authors should blind-annotate decay events on stratified trajectories, link interventions to action changes and outcomes, and stress-test stale, conflicting, and adversarial memories before irreversible tool actions.

What would most improve this paper

1. Release a complete, versioned reproduction package. Populate the advertised repository with code, license, prompts, configs, environment locks, task manifests, training-data provenance, raw trajectories, task-level results, exclusions, and exact commands. This is the highest-priority prerequisite for independent scrutiny.

2. Establish reliability and reconcile the reported results. Repeat all principal conditions, add paired uncertainty and tests, expose seed and exclusion accounting, and regenerate Tables 1–2 from versioned artifacts with a clear explanation of any differing evaluation populations.

3. Isolate memory from auxiliary deliberation. Use matched token and model budgets in a factorial bank/no-bank × selective/always/silent experiment, with critic, self-reflection, random-reminder, and no-op controls. This would directly test whether structured selective memory is the active ingredient.

4. Quantify the deployment tradeoff. Log interventions, calls, tokens, latency, dollar cost, bank size, failures, and harmful reminders; compare event-driven and periodic triggers; and report quality–cost Pareto frontiers.

5. Measure the proposed construct and its safety boundaries. Create blinded behavioral-state-decay annotations, report intervention precision, recall, and causal outcome changes, and test stale or adversarial memories across broader models and web, research, embodied, and software-engineering tasks.
