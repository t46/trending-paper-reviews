DOT-MoE makes a strong case that balanced differentiable optimal transport is a natural fit for converting dense feed-forward layers into sparse MoE layers. Its most consequential strength is the coherence between the method and the bottleneck it targets: balanced neuron-to-expert assignment, output-aware alignment, and learned routing are evaluated across several LLM families. The most important improvement direction is auditability — public reproduction artifacts, uncertainty estimates, compute-normalized baseline reruns, and direct diagnostics for the soft-to-hard routing pathway would make the strongest claims much easier to verify.

Paper overview

DOT-MoE studies how to convert pretrained dense LLM feed-forward layers into sparse Mixture-of-Experts layers without retraining a model from scratch. The paper argues that prior MoEfication methods rely heavily on heuristic neuron clustering or random splitting, then proposes a differentiable optimal transport formulation for balanced neuron-to-expert assignment.

The method uses Sinkhorn-Knopp iterations to learn balanced assignments and straight-through estimators to connect soft optimization with hard expert structure. It jointly learns token-to-expert routing and uses an output-aware alignment objective so the converted MoE layer is trained to preserve dense FFN behavior rather than only proxy neuron similarity.

Empirically, the paper reports results across LLaMA-2, LLaMA-3, and Qwen2.5 families, comparing against structured pruning, heuristic dense-to-MoE conversion, and random-split baselines. The claimed picture is practical and promising: retain much of the dense model's behavior while lowering the active-parameter budget through a standard MoE architecture compatible with fused expert kernels.

Key strengths

Strong problem-method fit. Balanced optimal transport directly targets the central bottleneck in dense-to-MoE conversion: assigning dense FFN neurons to capacity-limited experts without collapsing utilization or relying only on ad hoc clustering. The formulation is well aligned with the structure of the problem.

Output-aware alignment objective. The method optimizes toward preserving dense FFN output behavior, which is more relevant than matching only static neuron similarities. This makes the conversion objective better connected to downstream model quality.

Broad empirical validation. The experiments span multiple model families, pruning baselines, dense-to-MoE baselines, and ablations. That breadth makes the contribution more persuasive than a single-backbone efficiency result.

Confound-aware appendix evidence. Same-granularity comparisons, LTE comparison, attention extension, and a larger 32B appendix case all help address obvious alternative explanations. These additions make the empirical story more robust.

Practical serving alignment. The final converted model remains a standard MoE architecture, which matters for deployment because it can use existing expert-parallel and fused-kernel infrastructure rather than requiring a bespoke runtime.

Key concerns and constructive suggestions

Reproducibility is the largest current gap. For an implementation-heavy method paper, the absence of public code, exact configurations, data recipes, and evaluation commands materially limits independent validation. To address this, the authors should release the conversion code, training scripts, configs, environment files, benchmark commands, and model-specific recipes needed to reproduce the main tables.

Statistical confidence is underdeveloped. The reported tables appear to rely on point estimates, leaving unclear whether smaller differences between DOT-MoE and competing methods are stable. To address this, the authors should run at least several seeds for smaller models and key ablations, then report confidence intervals or standard deviations alongside the main results.

Compute-normalized comparisons need sharper control. DOT-MoE uses alignment plus fine-tuning, while some baselines may use different data, token budgets, hardware, tuning effort, or prior-reported numbers. To address this, the authors should rerun CMoE, LLaMA-MoE-v2, LTE, and selected pruning baselines under matched data, tokens, hardware, active-parameter budgets, and tuning protocols.

Rounding and straight-through behavior need direct diagnostics. The core method depends on a soft optimal-transport assignment becoming a hard expert structure, with STE gradients bridging the mismatch. To address this, the paper should report soft/hard assignment disagreement, expert utilization entropy, Sinkhorn convergence behavior, and sensitivity to initialization, temperature, and rounding choices.

Deployment evidence is still narrow. Compatibility with fused expert kernels is useful, but throughput alone does not fully characterize serving value. To address this, the authors should add end-to-end latency, memory footprint, energy, batch-size sensitivity, and serving-stack comparisons that separate algorithmic sparsity from runtime implementation effects.

What would most improve this paper

1. Release reproduction artifacts. Publish the model-conversion code, training and evaluation scripts, exact configs, environment files, and data recipe. This is the highest-leverage improvement because the method's value depends on implementation details that outside readers currently cannot audit.

2. Add statistical uncertainty. Run repeated seeds for the smaller model settings and key ablations, then report variation. This would clarify which improvements are robust and which should be treated as directional.

3. Standardize compute-normalized baseline reruns. Reevaluate the strongest competing MoEfication and pruning baselines under matched tokens, data, hardware, active-parameter budgets, and tuning effort. This would make the comparative claims cleaner.

4. Add routing and assignment stability diagnostics. Track soft-to-hard mismatch, utilization entropy, Sinkhorn convergence, and sensitivity to initialization or temperature. These measurements would directly test the pathway that makes DOT-MoE distinctive.

5. Extend practical serving measurements. Add latency, memory, energy, and batch-size studies across realistic serving setups. This would connect the conversion method more tightly to the inference-efficiency motivation.
