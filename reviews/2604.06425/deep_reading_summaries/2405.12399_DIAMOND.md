## Paper Overview

- **Title**: Diffusion for World Modeling: Visual Details Matter in Atari
- **Authors**: Eloi Alonso, Adam Jelley, Vincent Micheli, Anssi Kanervisto, Amos Storkey, Tim Pearce, François Fleuret
- **Year**: 2024 (May, revised)
- **Venue**: NeurIPS 2024 Spotlight

## Introduction Findings (Sections 1–2)

### Problem Statement
> "Recent world models predominantly operate on sequences of discrete latent variables to model environment dynamics. However, this compression into a compact discrete representation may ignore visual details that are important for reinforcement learning."
> — Abstract

### Claimed Contributions
1. "introduce diamond (DIffusion As a Model Of eNvironment Dreams), a reinforcement learning agent trained in a diffusion world model" (Abstract)
2. "analyze the key design choices that are required to make diffusion suitable for world modeling" (Abstract)
3. "diamond achieves a mean human normalized score of 1.46 on the competitive Atari 100k benchmark; a new best for agents trained entirely within a world model" (Abstract)
4. CS:GO interactive game engine extension (project page)

### Claimed Gaps in Prior Work
Prior methods like IRIS use discrete latent representations that lose visual details. DDPM-based diffusion is too slow and unstable for world modeling. DIAMOND proposes the EDM formulation as a solution.

## Method Section Findings (Section 3) — CRITICAL

### Overall Approach
DIAMOND uses an EDM-formulated diffusion model with U-Net 2D architecture, conditioned on past observations and actions via adaptive group normalization and channel concatenation. The model autoregressively generates next game frames, trained alongside an RL agent using the generated "imagination" as a training environment.

### Key Technical Components

#### Component 1: Diffusion World Model Architecture (Section 3.1)
> "We use a standard U-Net 2D for the vector field F_theta (Ronneberger et al., 2015)"
> — Section 3.1

EDM parameterization:
> "D_theta(x^{t+1}_tau, y^t_tau) = c_skip^tau * x^{t+1}_tau + c_out^tau * F_theta(c_in^tau * x^{t+1}_tau, y^t_tau)"
> — Section 3.1, Eq. 6

**Technical Details**:
- U-Net 2D (not 3D/video diffusion)
- EDM formulation enables stability with minimal denoising steps
- 4.4M parameters (Atari version)
- 381M parameters (CS:GO version with upsampler)

#### Component 2: Action Conditioning (Section 3.1)
> "we input actions through adaptive group normalization layers (Zheng et al., 2020) in the residual blocks"
> — Section 3.1

> "a buffer of L past observations and actions that we use to condition the model. We concatenate these past observations to the next noisy observation channel-wise"
> — Section 3.1

**Technical Details**:
- Actions: fed through adaptive group normalization (AdaGN) in residual blocks
- Observations: channel-wise concatenation to noisy next observation
- Context buffer: L past frames + L past actions

#### Component 3: Autoregressive Frame Generation (Section 2.3, Figure 1)
> "starting from an initial noisy sample x^t_T, we simulate a reverse noising process {x^t_tau}_{tau=T}^{tau=0} by repeatedly calling D_theta"
> — Section 2.3

> "The imagination procedure is autoregressive in that the predicted observation x^t_0 and the action a_t become part of the conditioning for the next time step"
> — Section 2.3

**Technical Details**:
- 3 denoising steps per frame (balance between quality and speed)
- 1 step: blurry for stochastic events; 3 steps: better mode selection
- EDM enables stability even at 1 NFE; DDPM fails at low NFE

#### Component 4: EDM vs. DDPM Design Choice (key finding)
> "EDM based diffusion world models produce stable trajectories even for 1 denoising step, while DDPM suffers from compounding error"
> — Project page

**Technical Details**:
- DDPM becomes unstable due to autoregressive error accumulation at low NFE
- EDM + skip connections allows stable 3 NFE per frame (vs. 16 NFE for IRIS)
- This parallels NC_CLIGen/NC_GUIWorld's open-loop vs. closed-loop stability challenge (CFR-004)

#### Component 5: RL Training Pipeline (Section 3.2)
- Separate reward/termination model (CNN-LSTM)
- Policy trained with REINFORCE + value baseline + lambda-returns
- Cycle: collect data → train world model → train agent in imagination (Algorithm 1)

### CS:GO Extension (Interactive Game Engine — Project Page)
> "The diffusion model functions as an interactive neural game engine where users provide actions, and the model autoregressively generates subsequent frames, enabling real-time gameplay within the learned environment simulation."
> — Project page

**Technical Details**:
- 87 hours of human CS:GO gameplay (no RL)
- Two-stage pipeline: low-resolution dynamics + upsampler
- ~10 FPS on RTX 3090
- Fixed dataset (no online RL data collection)
- Stochastic sampling for upsampler visual quality

### Method Overlap Assessment
**Overlap Level**: Medium — different backbone (U-Net vs. DiT), same core paradigm

**DIAMOND approach**: U-Net 2D diffusion model, action conditioning via AdaGN layers, channel-concatenated past frames, 3-step EDM.

**NC_GUIWorld approach**: Wan2.1 DiT diffusion transformer, four action injection modes, fine-tuning pretrained model, open-loop rollout.

**Key differences**:
- DIAMOND is trained from scratch on game data; NC_GUIWorld fine-tunes a pretrained video model
- DIAMOND uses U-Net 2D (per-frame); NC_GUIWorld uses 3D DiT (video)
- DIAMOND shows closed-loop interactive use for both RL and human play; NC_GUIWorld only tested open-loop (CFR-004)
- DIAMOND's CS:GO extension demonstrates human-interactive game engine capability — a direct precedent for NC_GUIWorld's claim of "interactive computer simulation"
