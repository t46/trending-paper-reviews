## Method Section Findings (Section 4) — CRITICAL

### Overall Approach
Add lightweight two-linear-layer modules to each transformer block of the pretrained video model, implementing scale-shift modulation from action embeddings. Optimize only the added parameters (and optionally the action embeddings) while keeping pretrained weights frozen. Apply motion-reinforced loss to emphasize dynamic regions.

### Key Technical Components

#### Component 1: Action-Conditioned Module Architecture (Section 4.1)
> "a lightweight add-on module, consisting of two linear layers within each transformer block"
> — Section 4.1

Scale-shift operation on frame embeddings:
> "x^i = x^i + FFN(LayerNorm(x^i) × (1+alpha^i) + beta^i)"

where alpha^i and beta^i are scale and shift parameters derived from action embeddings c^i.

**Technical Details**:
- Two linear layers only (minimal parameter overhead)
- Applies scale-shift modulation inside each transformer block
- Works for both spatial-temporal transformer blocks (diffusion models) and LLM blocks (autoregressive)
- Action representation:
  - Discrete: language templates mapping integer actions to semantic text descriptions, processed by pretrained text encoder
  - Continuous: trainable linear action embedder

#### Component 2: Motion-Reinforced Loss (Section 4.2)
> "omega_{i+1} = c^{Softmax(|x_{i+1}-x_i|)}"
> "L_motion = L_prev * omega"
> — Section 4.2

**Technical Details**:
- Computes inter-frame differences (motion mask)
- Weights pixels based on change magnitude
- Inherently attenuates static background elements
- For diffusion: weights MSE loss; for transformers: weights cross-entropy loss
- Motivation: "RL agents predominantly learn from action-induced state changes rather than static visual elements"

#### Component 3: Supported Model Types (Section 5.1)
Two base models tested:
1. **Open-Sora** (diffusion-based, rectified flow, spatial-temporal transformers)
2. **iVideoGPT** (autoregressive, LLaMA-based)

> "DWS demonstrates its versatility as a universal method...seamlessly integrated into various architectures"
> — Section 5.1

#### Component 4: Training Procedure
- Only the added two-linear-layer modules optimized (frozen pretrained weights)
- Motion-reinforced loss applied throughout training
- "Low training costs" claimed (concrete numbers not visible in fetched content)

### Key Equations
- Scale-shift: x^i = x^i + FFN(LayerNorm(x^i) × (1+alpha^i) + beta^i) — Section 4.1
- Motion weight: omega_{i+1} = c^{Softmax(|x_{i+1}-x_i|)} — Section 4.2
- Motion loss: L_motion = L_prev * omega — Section 4.2

## Experiments Summary (Section 5)

### Datasets Used
- BAIR robot dataset (continuous actions, 64×64)
- Procgen: Coinrun, Ninja (discrete actions, 64×64)
- Atari: Breakout, Battle Zone (discrete actions, 84×84)

### Baselines Compared
- AVID (Rigter et al., 2024) — mentioned as concurrent work
- Yu et al. (2025) — another adapter approach

### Key Results
- Atari with iVideoGPT+DWS: FVD 9.1 vs. 11.5 baseline, PSNR 43.4 vs. 39.1, SSIM 98.2 vs. 97.3
- MBRL on Breakout: "7× performance gain compared to SOTA methods" (Section 5.2)

## Relevance to Target Paper (Neural Computers)

### Problem Overlap Assessment
**Overlap Level**: High — directly parallel technical problem

**Evidence**:
> "converting pre-trained video generative models into world simulators" with action conditioning
> — Section 4

**Comparison**: NC_GUIWorld converts a pretrained video diffusion model (Wan2.1) into an action-conditioned GUI simulator by adding action injection modules. DWS does the same for games/robotics. The technical operation is identical: add lightweight action-conditioning to pretrained video model.

### Method Overlap Assessment
**Overlap Level**: High — direct methodological parallel

**DWS approach**: Two linear layers per transformer block implementing scale-shift modulation from action embeddings (alpha^i, beta^i).

**NC_GUIWorld approach** (from project blog): Four injection modes including external (input-side), contextual (token merging), residual (side branch), internal (cross-attention in blocks). The "internal" mode adds cross-attention sub-layers inside each DiT block — conceptually very similar to DWS's per-block modulation approach.

**Key differences**:
- DWS keeps pretrained weights frozen; NC_GUIWorld fine-tunes the full model
- DWS uses scale-shift modulation (two linear layers); NC_GUIWorld uses cross-attention (more parameters, more expressivity)
- DWS supports both diffusion and autoregressive; NC_GUIWorld focused on DiT
- DWS applies to games/robotics; NC_GUIWorld applies to OS/GUI
