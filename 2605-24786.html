CONF-KV: Confidence-Aware KV Cache Eviction with Mixed-Precision Storage for Long-Horizon LLM Inference

CONF-KV's strongest contribution is the confidence-gated control signal: it uses information already present in the next-token distribution to adapt KV retention over decoding time, and the paper supports that signal with useful mechanism evidence and broad systems-oriented evaluation. The most important improvement direction is auditability — releasing runnable artifacts, comparing against newer adaptive KV methods, and stress-testing confidently wrong or rare-entity cases would make the central claims much easier to verify and reuse.

Paper overview

CONF-KV addresses a practical long-horizon inference bottleneck: as generation length grows, the key-value cache becomes a dominant GPU-memory consumer and increases the cost of attention. The paper proposes a cache manager that converts the current next-token distribution into a scalar confidence score, then uses that score to choose a per-step retention budget.

The method ranks tokens by accumulated attention mass plus recency while protecting a recent window. It combines this policy with blockwise online-softmax attention, mixed FP16/INT8 KV storage, and a pyramidal per-layer variant, aiming for near-sliding-window memory behavior while preserving much of full-cache quality.

The evaluation covers perplexity, memory footprint, Needle-in-a-Haystack retrieval, VisualWebArena, latency, throughput, profiling, and ablations across several model families. The paper is technically coherent and timely, but because the contribution depends on custom cache management, compaction, quantization, and kernel behavior, independent artifact access matters more than it would for a purely conceptual method.

Key strengths

Clear, timely systems problem. KV-cache growth is a first-order constraint for long-context serving, multi-turn assistants, and agentic workloads. The paper anchors the method in concrete serving pressure rather than treating cache compression as an isolated benchmark exercise.

Simple training-free control signal. CONF-KV uses logits already produced during decoding to estimate confidence, requiring no model retraining or weight modification. That makes the idea composable with existing inference stacks and easier to reason about than learned or task-specific controllers.

Mechanistic validation of the main assumption. The paper does more than report end-task metrics: it uses KL-ablation evidence to support the claim that lower confidence is associated with higher context sensitivity. This is important because the method's value depends on confidence being informative about when cached context matters.

Broad systems-oriented evaluation surface. The experiments span quality, retrieval, web-agent behavior, memory, latency, throughput, ablations, and profiling. That breadth helps connect the algorithmic idea to deployment-relevant behavior rather than only showing perplexity preservation.

Transparent limitations discussion. The paper explicitly discusses confidently wrong rare-entity failures, high-temperature entropy saturation, short-context no-op behavior, MLP-dominated workloads, and friction with paged serving layouts. That candor makes the method's likely operating envelope clearer.

Key concerns and constructive suggestions

Reproducibility is currently incomplete. The work is implementation-based, but the paper states that code will be released later, leaving core memory, latency, compaction, quantization, and benchmark-wrapper claims unauditable. To address this, the authors should release an anonymized runnable artifact with kernels, configs, benchmark wrappers, and scripts for the main figures and tables.

Core systems claims need artifact-level verification. Written hyperparameters help, but custom compaction, mixed-precision storage, tiled attention, and cache-management details can materially change reported results. To address this, the artifact should include a paper-to-command mapping that lists the exact config, expected output, and compute environment for each central result.

The baseline set should be updated for the current KV-compression landscape. The included comparisons are relevant, but newer or concurrent adaptive methods such as KVzip, LKV, AhaKV, EasyKV/RoCo, and simple norm-based diagnostics are missing. To address this, the authors should add at least two implementable recent baselines and clearly document any comparisons that are infeasible because of missing code or incompatible assumptions.

Uncertainty reporting is uneven around close comparisons. Some results use repeated runs, but important perplexity tables and small margins against strong baselines lack prompt-level or document-level uncertainty. To address this, the paper should report bootstrap confidence intervals or document-level variance for perplexity, and tabulate exact standard deviations for retrieval, web-agent, and throughput figures.

The confidence proxy has important failure modes. The authors acknowledge rare-entity cases where the model can be confidently wrong before retrieving the needed evidence, which directly targets the method's main signal. To address this, the evaluation should add rare-entity, high-temperature, and overconfidence slices, plus detector or fallback logic for cases where confidence is poorly calibrated.

Production serving compatibility remains unproven. Token-level eviction can create partially dead blocks in paged serving layouts, so prototype contiguous compaction may not translate cleanly to vLLM-like deployment. To address this, the authors should implement block-coarsened eviction or sparse-mask variants and benchmark them under a paged-layout serving setup.

What would most improve this paper

1. Release a reproducibility artifact. Provide runnable code, kernels, configs, evaluation scripts, and benchmark wrappers so the memory, quality, latency, and throughput claims can be independently checked.

2. Map paper results to commands. Add a reproduction README that links each main table and figure to the exact command, configuration, expected output, and compute requirement.

3. Add current adaptive KV baselines. Include recent implementable methods and discuss omitted methods explicitly, so CONF-KV's comparative position is clear in the fast-moving cache-compression literature.

4. Strengthen uncertainty reporting. Bootstrap over prompts or documents for perplexity and provide exact standard deviations for plotted retrieval, web-agent, and throughput categories.

5. Stress-test confidence failure slices. Evaluate rare-entity lookup, high-temperature sampling, and overconfidence cases, then add calibration guidance or fallback settings for regimes where confidence is unreliable.

6. Validate paged serving integration. Prototype block-level or sparse-mask variants under a paged serving layout to separate algorithmic benefit from integration overhead.
